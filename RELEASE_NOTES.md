# Server Setup - Релиз v2.0.1

## 🛡️ Описание

Модульный инструмент для первоначальной настройки, установки ПО и администрирования серверов под управлением Debian/Ubuntu.

Проект предоставляет интерактивное меню для автоматизации рутинных задач по настройке и администрированию сервера, следуя лучшим практикам информационной безопасности.

## 🚀 Возможности (текущая версия)

### ✅ Реализовано
*   **🔄 Обновление системы** — полный цикл обновления: `apt update`, `upgrade`, `dist-upgrade`, очистка кэша и удаление ненужных пакетов.
*   **🌐 Управление зеркалами APT** — быстрая замена репозиториев на Yandex Mirror, ручная настройка, создание бэкапов и восстановление конфигурации.

### 🔜 В планах (заглушки в меню)
*   Создание пользователя с ограниченными правами (sudo).
*   Настройка SSH: смена порта, отключение входа root, настройка SSH-ключей.
*   Базовая настройка фаервола (UFW / Firewalld).
*   Установка и базовая конфигурация Fail2ban.
*   Отключение ненужных системных служб.
*   Настройка автоматических обновлений (unattended-upgrades).
*   Дополнительные меры: 2FA, права доступа к конфигурационным файлам.

## 🚀 Установка и запуск

### Установка одной командой (рекомендуемый способ)
Скрипт автоматически скачивает репозиторий и устанавливает его в `/opt/server-setup`:

```bash
bash <(curl -Ls https://raw.githubusercontent.com/kewgenia/script/main/install.sh)
```

### Альтернативный способ: Клонирование и установка
Скрипт устанавливается в директорию `/opt/server-setup` и создает симлинк для быстрого доступа.

```bash
# Клонирование репозитория
git clone https://github.com/kewgenia/script.git /opt/server-setup

# Запуск установщика
cd /opt/server-setup
sudo bash install.sh
```

### Запуск
После установки скрипт доступен глобально:

```bash
sudo server-setup
# или
sudo /opt/server-setup/server-setup.sh
```

## 📂 Структура проекта

Проект использует модульную архитектуру с автоматической генерацией меню через манифесты:

*   `server-setup.sh` — главный скрипт (точка входа).
*   `install.sh` — скрипт установки в систему.
*   `modules/core/common.sh` — ядро: UI-хелперы, цвета, логирование.
*   `modules/security/` — модули безопасности (обновление, зеркала).
