# Server Setup - Релиз v2.1.0

## 🛡️ Описание

Модульный инструмент для первоначальной настройки, установки ПО и администрирования серверов под управлением Debian/Ubuntu.

Проект предоставляет интерактивное меню для автоматизации рутинных задач по настройке и администрированию сервера, следуя лучшим практикам информационной безопасности.

## 🚀 Возможности (текущая версия)

### ✅ Реализовано
*   **🔄 Обновление системы** — полный цикл обновления: `apt update`, `upgrade`, очистка кэша, удаление ненужных пакетов и проверка необходимости перезагрузки.
*   **🌐 Управление зеркалами APT** — быстрая замена репозиториев на Yandex Mirror, ручная настройка, создание бэкапов и восстановление конфигурации.
*   **🏗️ Архитектура** — модульная структура с автоматической генерацией меню через манифесты `@menu.manifest`.
*   **🎨 Улучшенный UI** — цветной вывод, прогресс-бары, информативные сообщения.
*   **📋 Логирование** — подробные логи в `/var/log/server-setup.log` с поддержкой уровней вывода.
*   **⚙️ Обработка ошибок** — перехват ошибок через `trap` с автоматической очисткой временных файлов.
*   **🌍 Определение окружения** — автоматическое определение типа виртуализации, провайдера и окружения.

### 🔜 В планах (заглушки в меню)
*   Создание пользователя с ограниченными правами (sudo).
*   Настройка SSH: смена порта, отключение входа root, настройка SSH-ключей.
*   Базовая настройка фаервола (UFW / Firewalld).
*   Установка и базовая конфигурация Fail2ban.
*   Отключение ненужных системных служб.
*   Настройка автоматических обновлений (unattended-upgrades).
*   Дополнительные меры: 2FA, права доступа к конфигурационным файлам.

## 🚀 Установка и запуск

### Установка одной командой (рекомендуемый способ)
Скрипт автоматически скачивает репозиторий и устанавливает его в `/opt/server-setup`:

```bash
bash <(curl -Ls https://raw.githubusercontent.com/kewgenia/script/main/install.sh)
```

После установки скрипт доступен как `server-setup` (если `/usr/local/bin` в PATH).

### Локальная установка
1. Скачайте репозиторий:
   ```bash
   git clone https://github.com/kewgenia/script.git
   cd script
   ```

2. Запустите скрипт установки:
   ```bash
   sudo ./install.sh
   ```

### Использование
- **Интерактивный режим:**
  ```bash
  sudo -E server-setup
  ```

- **Тихий режим (для автоматизации):**
  ```bash
  sudo -E server-setup --quiet
  ```

- **Автоматический режим:**
  ```bash
  sudo -E server-setup --auto
  ```

- **Справка:**
  ```bash
  sudo -E server-setup --help
  ```

## 📋 Логирование

- Логи сохраняются в `/var/log/server-setup_*.log`
- Бэкапы конфигураций в `/root/setup-backup_*`
- Отчеты генерируются в `/var/log/server-setup_report_*.txt`
