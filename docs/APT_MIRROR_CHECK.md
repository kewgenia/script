# Скрипт проверки и замены APT репозиториев

## Описание

`apt-mirror-check.sh` — это интерактивный скрипт для управления APT репозиториями в системах семейства Debian (Debian, Ubuntu, Linux Mint и другие). Скрипт позволяет проанализировать текущие источники обновлений, заменить их на зеркала Yandex для ускорения работы или настроить собственные зеркала вручную.

## Функционал

Скрипт выполняет следующие операции:

1. **Определение дистрибутива** — автоматическое определение типа системы (Debian, Ubuntu и др.) и кодового имени версии
2. **Анализ текущих репозиториев** — сбор информации из `/etc/apt/sources.list` и `/etc/apt/sources.list.d/`
3. **Создание резервных копий** — автоматическое сохранение текущей конфигурации перед внесением изменений
4. **Замена на зеркала Yandex** — быстрая настройка репозиториев на сервера Яндекса
5. **Ручная настройка зеркал** — возможность ввода пользовательских URL зеркал
6. **Восстановление из резервной копии** — откат изменений в случае необходимости
7. **Обновление списков пакетов** — проверка работоспособности новых репозиториев

## Особенности

- **Интерактивное меню** — удобный выбор действий через текстовый интерфейс
- **Цветной вывод** в терминал для лучшей визуализации процесса:
  - 🟢 Зелёный — успешные операции
  - 🟡 Жёлтый — предупреждения и запросы
  - 🔴 Красный — ошибки
  - 🔵 Синий — информационные сообщения
  - 🔵 Голубой — дополнительная информация
- **Полное логирование** всех операций в файл `/var/log/apt-mirror-check.log`
- **Безопасность** — автоматическое создание резервных копий перед любыми изменениями
- **Поддержка DEB822** — обнаружение файлов в новом формате (`.sources`)

## Требования

- Система семейства Debian (Debian, Ubuntu, Linux Mint, Pop!_OS и др.)
- Права суперпользователя (root)
- Установленные утилиты: `apt-get`, `tee`, `date`, `lsb_release` (опционально)

## Установка

1. Скачайте скрипт:
```bash
wget https://ваш-репозиторий/apt-mirror-check.sh
```

2. Сделайте скрипт исполняемым:
```bash
chmod +x apt-mirror-check.sh
```

## Использование

Запустите скрипт с правами суперпользователя:

```bash
sudo ./apt-mirror-check.sh
```

## Меню скрипта

При запуске скрипт отображает текущие репозитории и предлагает следующие действия:

```
========================================
 Меню управления репозиториями
========================================
1. Заменить репозитории на зеркала Yandex
2. Ввести свои зеркала вручную
3. Обновить списки пакетов
4. Восстановить из резервной копии
5. Показать текущие репозитории
0. Выход
========================================
```

### Описание пунктов меню

#### 1. Заменить репозитории на зеркала Yandex

Автоматически настраивает репозитории на зеркала Яндекса:
- Для Debian: `http://mirror.yandex.ru/debian/`
- Для Ubuntu: `http://mirror.yandex.ru/ubuntu/`
- Для Linux Mint: `http://mirror.yandex.ru/linuxmint-packages/`

Скрипт создаст резервную копию и перезапишет `/etc/apt/sources.list`, а также отключит сторонние репозитории в `sources.list.d`.

#### 2. Ввести свои зеркала вручную

Позволяет указать произвольный URL зеркала. Скрипт запросит:
- URL основного зеркала
- Нужно ли добавлять репозиторий security
- Кодовое имя дистрибутива (если не определено автоматически)

#### 3. Обновить списки пакетов

Выполняет `apt-get update` для проверки доступности настроенных репозиториев.

#### 4. Восстановить из резервной копии

Показывает список доступных резервных копий (сохраняются в `/etc/apt/backup-*`) и позволяет выбрать одну для восстановления.

#### 5. Показать текущие репозитории

Выводит список всех активных репозиториев из `sources.list` и `sources.list.d`.

## Резервное копирование

Перед любыми изменениями скрипт создаёт резервную копию в директории вида:
```
/etc/apt/backup-YYYYMMDD-HHMMSS/
```

Структура резервной копии:
```
/etc/apt/backup-20260429-181500/
├── sources.list.bak
└── sources.list.d/
    ├── some-repo.list
    └── another-repo.list.disabled
```

## Журнал логирования

Все операции скрипта записываются в файл `/var/log/apt-mirror-check.log` в формате:

```
[2026-04-29 18:15:00] Начато выполнение скрипта проверки репозиториев
[2026-04-29 18:15:01] Обнаружен дистрибутив: ubuntu (кодовое имя: jammy)
[2026-04-29 18:15:05] Создана резервная копия: /etc/apt/backup-20260429-181500
[2026-04-29 18:15:06] Репозитории заменены на зеркала Yandex
```

Просмотр журнала:
```bash
cat /var/log/apt-mirror-check.log
```

## Поддерживаемые дистрибутивы

Скрипт автоматически определяет и настраивает зеркала для:
- **Debian** (и производные)
- **Ubuntu** (и производные)
- **Linux Mint**
- **Pop!_OS**
- **elementary OS**

Для других дистрибутивов используется общее зеркало Debian, но можно настроить вручную через пункт меню 2.

## Возможные проблемы и решения

### Ошибка "Permission denied"

**Решение:** Убедитесь, что скрипт запущен с правами root:
```bash
sudo ./apt-mirror-check.sh
```

### Ошибка при обновлении списков пакетов

Если после замены репозиториев `apt-get update` выдаёт ошибки:

1. Проверьте правильность кодового имени дистрибутива:
   ```bash
   lsb_release -cs
   ```

2. Убедитесь, что зеркало доступно:
   ```bash
   ping mirror.yandex.ru
   ```

3. Восстановитесь из резервной копии через пункт меню 4

### Нехватка места на диске

Если резервных копий накопилось много, их можно удалить:
```bash
sudo rm -rf /etc/apt/backup-*
```

### Файлы в формате DEB822 (.sources)

Скрипт обнаруживает файлы в новом формате DEB822, но не обрабатывает их автоматически. Если у вас есть такие файлы, настройте их вручную или отключите, переименовав с добавлением расширения `.disabled`.

## Безопасность

⚠️ **Важно:**
- Скрипт создаёт резервные копии перед любыми изменениями — не удаляйте их сразу после настройки
- Проверьте работоспособность репозиториев через пункт меню 3 перед выходом
- На production-серверах рекомендуется сначала протестировать настройки на staging-среде
- После замены репозиториев рекомендуется выполнить полное обновление системы (например, с помощью `debian-update.sh`)
