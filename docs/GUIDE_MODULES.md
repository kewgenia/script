# Гайд по модулям: как писать и как встраивать

Этот документ объясняет высокоуровневую структуру модулей и как они интегрируются в меню.

**Все "железные" стандарты оформления кода, использования UI-хелперов и взаимодействия с системой вынесены в отдельный документ:**

**➡️ [`STYLE_GUIDE.md`](STYLE_GUIDE.md)**

Перед чтением этого гайда **настоятельно рекомендуется** ознакомиться со `STYLE_GUIDE.md`.

---

## 1. Где живут модули и как они подключаются

- Все модули лежат в каталоге `modules/`, сгруппированные по поддиректориям:
  - `modules/core/` — ядро системы
  - `modules/security/` — модули безопасности
  - и т.д.
- Главный скрипт `server-setup.sh` предоставляет функцию-роутер:
  - `run_module <module_path> <function> [args...]`
    - она делает `source` указанного файла модуля и вызывает функцию.
- Любой модуль **обязан** рассчитывать на то, что перед его вызовом уже загружены:
  - `config/server.conf` (глобальные константы);
  - `modules/core/common.sh` (цвета, хелперы меню, `run_cmd`, `safe_read`, `ask_yes_no` и т.д.).

**Никогда** не запускаем модули напрямую (`bash modules/security/my_feature.sh`) — только через `run_module`.

## 2. Интеграция модуля в меню: Архитектура «Манифест меню»

В проекте используется **полностью автоматическая генерация меню**, основанная на "манифестах". Вам больше не нужно редактировать центральные файлы, чтобы добавить пункт в меню.

Просто добавьте в начало вашего `.sh` файла модуля специальный блок-комментарий `@menu.manifest`, содержащий один или несколько пунктов `@item`:

```bash
#
# @menu.manifest
#
# @item( main | 4 | 💿 Моя Фича | show_my_feature_menu | 40 | 40 | Установка и управление )
#
```

### Формат `@item`

Каждый пункт меню описывается одной строкой со следующими 7 полями, разделенными `|`:

`@item( PARENT | KEY | TITLE | FUNCTION | ORDER | GROUP | DESCRIPTION )`

1.  **PARENT**: ID родительского меню. `main` для главного меню, или ID другого меню для создания подменю.
2.  **KEY**: Клавиша для вызова (например, `4` или `s`).
3.  **TITLE**: Название пункта меню, которое увидит пользователь. Можно использовать переменные цвета (например, `${C_BLUE}Имя`).
4.  **FUNCTION**: Имя функции в вашем модуле, которая будет вызвана.
5.  **ORDER**: Число для сортировки пунктов (чем меньше, тем выше).
6.  **GROUP**: Число для визуальной группировки. Пункты с одинаковым `GROUP` отображаются вместе, без пустой строки между ними.
7.  **DESCRIPTION**: (Опционально) Краткое описание, которое выводится под названием пункта.

При запуске `server-setup.sh` скрипт `modules/core/menu_generator.sh` сканирует все модули, находит эти манифесты и сам строит все меню в памяти.

## 3. Каркас современного модуля

В новой архитектуре модуль, который просто *отображает* подменю, предельно прост. Ему не нужно самому рендерить пункты.

```bash
#!/bin/bash
# ============================================================ #
# ==                 МОДУЛЬ "МОЯ ФИЧА"                      == #
# ============================================================ #
#
# @menu.manifest
# @item( main | x | Моя новая фича | show_my_feature_menu | 55 | 50 | Демонстрация нового модуля )
# @item( my_feature | 1 | Сделать что-то | _my_feature_do_something | 10 | 10 | )
# @item( my_feature | 2 | Сделать опасное | _my_feature_ask_danger | 20 | 10 | )
#
[[ "${BASH_SOURCE[0]}" == "${0}" ]] && exit 1 # Защита от прямого запуска

# === ЛОКАЛЬНЫЕ ХЕЛПЕРЫ (private-функции) =====================

_my_feature_do_something() {
    ok "Сделано!"
    wait_for_enter
}

_my_feature_ask_danger() {
    if ! ask_yes_no "Это опасное действие. Вы уверены?"; then
        info "Отменено пользователем."
        wait_for_enter
        return
    fi
    err "БУМ! (шутка)"
    wait_for_enter
}


# === ГЛАВНОЕ МЕНЮ МОДУЛЯ =====================================

show_my_feature_menu() {
    # Это меню является "родителем" для других пунктов.
    # Его ID для PARENT в других манифестах будет "my_feature".
    local menu_id="my_feature" 
    
    enable_graceful_ctrlc
    while true; do
        clear
        menu_header "🎉 Моя новая фича"
        printf_description "Это меню демонстрирует, как работает render_menu_items."

        # Генератор сам найдет и отрисует все пункты,
        # у которых в манифесте указано @item( my_feature | ... )
        render_menu_items "$menu_id"

        printf_menu_option "b" "Назад"

        local choice
        choice=$(safe_read "Ваш выбор: ") || break
        
        if [[ "$choice" == "b" || "$choice" == "B" ]]; then
            break
        fi

        # Получаем и выполняем действие, связанное с выбранной клавишей
        local action
        action=$(get_menu_action "$menu_id" "$choice")
        
        if [[ -n "$action" ]]; then
            eval "$action"
        else
            printf_error "Нет такого пункта." && sleep 1
        fi
    done
    disable_graceful_ctrlc
}
