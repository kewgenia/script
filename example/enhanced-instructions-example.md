# Пример расширенного файла .instructions

Этот файл содержит пример того, как можно расширить `.instructions` для более эффективной работы с проектом визитки.

## Как использовать

1. Скопируйте содержимое блока ниже (между `---`)
2. Замените содержимое файла `.instructions` в корне проекта
3. Настройте параметры под свои нужды

---

## Рекомендуемое содержимое для .instructions

```bash
# Инструкции для ассистента
# Этот файл содержит настройки и инструкции для работы с проектом

# ============================================
# ОСНОВНЫЕ НАСТРОЙКИ
# ============================================

# Язык ответов
RESPOND_IN_RUSSIAN=true
LANGUAGE=ru

# ============================================
# КОНТЕКСТ ПРОЕКТА
# ============================================

# Тип проекта
PROJECT_TYPE=static_website
PROJECT_DESCRIPTION=Сайт-визитка системного администратора и DevOps-инженера

# Технологический стек
TECH_STACK=HTML5,CSS3,Docker,Caddy,Cloudflare

# ============================================
# ИНСТРУКЦИИ ДЛЯ РЕЖИМОВ
# ============================================

# Инструкции для режима Architect
[mode:architect]
- Создавай планы в директории plans/
- Используй диаграммы Mermaid для визуализации архитектуры
- Всегда создавай todo list для задач из 2+ шагов
- Учитывай инфраструктуру: Docker, Caddy, Cloudflare DNS-01
- Предлагай решения, совместимые с текущим стеком

# Инструкции для режима Code
[mode:code]
- Соблюдай структуру проекта: site/ для контента, корень для конфигов
- При редактировании HTML/CSS используй современные стандарты
- Для Docker/Caddy конфигов добавляй комментарии на русском
- Проверяй корректность YAML для docker-compose.yml
- Используй 2 пробела для отступов в конфигурационных файлах

# Инструкции для режима Debug
[mode:debug]
- Проверяй логи Docker: docker compose logs -f caddy
- При проблемах с SSL проверяй настройки Cloudflare API токена
- Тестируй конфигурацию Caddy перед применением
- Проверяй доступность портов 80 и 443

# Инструкции для режима Ask
[mode:ask]
- Объясняй концепции инфраструктуры (Docker, Caddy, Let's Encrypt)
- Приводи примеры конфигураций для похожих задач
- Давай ссылки на официальную документацию

# ============================================
# СПЕЦИФИЧНЫЕ ПРАВИЛА ПРОЕКТА
# ============================================

# Структура файлов
- Статические файлы сайта: site/
- Конфигурация Caddy: Caddyfile
- Docker настройки: Dockerfile, docker-compose.yml
- Переменные окружения: .env (не коммитить!), .env.example
- Планы и документация: plans/

# Безопасность
- НИКОГДА не коммить файл .env в git
- API токены должны быть только в .env
- Проверяй .gitignore перед коммитом

# Docker и Caddy
- Переменные в Caddyfile используют синтаксис {$VAR_NAME}
- Статические файлы монтируются как read-only (ro)
- Сертификаты хранятся в volume caddy_data

# ============================================
# ШАБЛОНЫ ПРОМПТОВ
# ============================================

# Быстрые команды (можно использовать как примеры)
# "Обнови сайт: добавь секцию [НАЗВАНИЕ] в site/index.html"
# "Проверь конфигурацию Caddyfile на ошибки"
# "Создай план по добавлению [ФУНКЦИЯ]"
# "Объясни, как работает DNS-01 challenge в Caddy"

# ============================================
# КОНТАКТНАЯ ИНФОРМАЦИЯ
# ============================================

# Контакты владельца проекта (для использования в сайте)
CONTACT_GITHUB=https://github.com/kewgenia
# Другие контакты можно добавить в .env и использовать через {$CONTACT_*}
```

## Пояснения к настройкам

### 1. Основные настройки
- `RESPOND_IN_RUSSIAN=true` — заставляет ассистента отвечать на русском
- `LANGUAGE=ru` — дополнительный параметр для языка

### 2. Контекст проекта
Помогает ассистенту быстрее понимать суть проекта без дополнительных вопросов.

### 3. Инструкции для режимов `[mode:...]`
Специфичные правила, которые применяются только в определенных режимах:
- **architect** — планирование и архитектура
- **code** — написание кода
- **debug** — отладка
- **ask** — вопросы и объяснения

### 4. Специфичные правила
Правила безопасности и структуры проекта, которые ассистент будет соблюдать.

### 5. Шаблоны промптов
Примеры быстрых команд, которые можно использовать (закомментированы, чтобы не влияли на парсинг).

## Как применить

### Вариант 1: Через режим Code
Попросите: "Переключись в режим code и обнови файл .instructions, используя содержимое из plans/enhanced-instructions-example.md"

### Вариант 2: Вручную
1. Откройте файл `.instructions` в VSCode
2. Скопируйте содержимое из блока кода выше (без обертки ```bash)
3. Вставьте в файл `.instructions`
4. Сохраните файл

---

## Дополнительные советы

1. **Переменные окружения**: Используйте `.env` для чувствительных данных, а `.instructions` — для настроек поведения ассистента

2. **Комментарии**: В `.instructions` можно использовать `#` для комментариев

3. **Секции**: Используйте разделители `===` для визуальной структуры (они игнорируются парсером)

4. **Динамическое обновление**: Файл `.instructions` читается при каждом запросе, поэтому изменения применяются сразу же

5. **Специфичные инструкции**: Можно добавлять любые инструкции, специфичные для вашего проекта
