# Скрипты для систем семейства Debian

## Описание

Коллекция скриптов для автоматизации администрирования систем семейства Debian (Debian, Ubuntu, Linux Mint и другие производные).

### Доступные скрипты:

1. **[`debian-update.sh`](DEBIAN_UPDATE.md)** — автоматизация полного цикла обновления системы (подробнее в [DEBIAN_UPDATE.md](DEBIAN_UPDATE.md))
2. **[`apt-mirror-check.sh`](APT_MIRROR_CHECK.md)** — управление APT репозиториями и зеркалами (подробнее в [APT_MIRROR_CHECK.md](APT_MIRROR_CHECK.md))

## debian-update.sh

Подробная документация: **[DEBIAN_UPDATE.md](DEBIAN_UPDATE.md)**

### Описание

`debian-update.sh` — это скрипт для автоматизации процесса обновления систем семейства Debian. Скрипт выполняет полный цикл обновления: от обновления списков пакетов до очистки системы от устаревших файлов.

### Функционал

Скрипт выполняет следующие операции:

1. **Проверка прав root** — убедиться, что скрипт запущен с правами суперпользователя
2. **Обновление списков пакетов** (`apt-get update`) — получение актуальной информации о доступных пакетах
3. **Обновление установленных пакетов** (`apt-get upgrade -y`) — обновление всех установленных пакетов до последних версий
4. **Дистрибутивное обновление** (`apt-get dist-upgrade -y`) — обновление с учётом изменения зависимостей (установка/удаление пакетов при необходимости)
5. **Удаление ненужных пакетов** (`apt-get autoremove -y`) — удаление пакетов, которые больше не требуются системе
6. **Очистка кэша пакетов** (`apt-get clean`) — удаление загруженных .deb файлов из кэша
7. **Удаление устаревших пакетов** (`apt-get autoclean -y`) — удаление только устаревших загруженных пакетов
8. **Проверка необходимости перезагрузки** — определение, требуется ли перезагрузка системы для применения обновлений

### Особенности

- **Цветной вывод** в терминал для лучшей визуализации процесса:
  - 🟢 Зелёный — успешные операции
  - 🟡 Жёлтый — предупреждения
  - 🔴 Красный — ошибки
  - 🔵 Синий — информационные сообщения

- **Полное логирование** всех операций в файл `/var/log/debian-update.log`

- **Обработка ошибок** — скрипт завершает выполнение при критических ошибках с соответствующим сообщением

- **Отметки времени** — фиксация времени начала и окончания обновления

### Требования

- Система семейства Debian (Debian, Ubuntu, Linux Mint, Pop!_OS и др.)
- Права суперпользователя (root)
- Установленные утилиты: `apt-get`, `tee`, `date`

### Установка

1. Скачайте скрипт:
```bash
wget https://ваш-репозиторий/debian-update.sh
```

2. Сделайте скрипт исполняемым:
```bash
chmod +x debian-update.sh
```

### Использование

#### Базовое использование

Запустите скрипт с правами суперпользователя:

```bash
sudo ./debian-update.sh
```

#### Автоматическое обновление (cron)

Для регулярного автоматического обновления добавьте задачу в cron:

```bash
sudo crontab -e
```

Добавьте строку для еженедельного обновления (например, по воскресеньям в 3:00):

```
0 3 * * 0 /путь/к/debian-update.sh
```

#### Запуск в фоне

Для запуска скрипта в фоновом режиме:

```bash
sudo nohup ./debian-update.sh > /dev/null 2>&1 &
```

### Журнал логирования

Все операции скрипта записываются в файл `/var/log/debian-update.log` в формате:

```
[2026-04-28 10:30:15] Начато обновление системы
[2026-04-28 10:30:20] Обновление списков пакетов...
[2026-04-28 10:30:45] Списки пакетов успешно обновлены.
...
```

Просмотреть журнал:

```bash
cat /var/log/debian-update.log
```

Просмотреть последние записи:

```bash
tail -n 50 /var/log/debian-update.log
```

Мониторинг в реальном времени:

```bash
tail -f /var/log/debian-update.log
```

### Перезагрузка системы

После завершения работы скрипт проверяет необходимость перезагрузки. Если перезагрузка требуется, вы увидите сообщение:

```
⚠️  Требуется перезагрузка для завершения обновления.
```

Для перезагрузки выполните:

```bash
sudo reboot
```

Или используйте автоматическую перезагрузку (добавьте в конец скрипта):

```bash
shutdown -r now
```

### Возможные проблемы и решения

#### Ошибка "Permission denied"

**Решение:** Убедитесь, что скрипт запущен с правами root:
```bash
sudo ./debian-update.sh
```

#### Ошибка "Command not found"

**Решение:** Убедитесь, что скрипт исполняемый:
```bash
chmod +x debian-update.sh
```

#### Блокировка apt (dpkg frontend lock)

**Решение:** Дождитесь завершения других операций apt или перезагрузите систему:
```bash
sudo killall apt apt-get
sudo rm /var/lib/dpkg/lock-frontend
sudo dpkg --configure -a
```

#### Нехватка места на диске

**Решение:** Очистите диск перед обновлением:
```bash
sudo apt-get autoremove -y
sudo apt-get clean
sudo rm -rf /var/cache/apt/archives/*
```

### Безопасность

⚠️ **Важно:**
- Всегда проверяйте, какие обновления будут установлены, перед запуском скрипта на production-серверах
- Создавайте резервные копии важных данных перед массовым обновлением
- Тестируйте обновления на staging-серверах перед применением на production
- Скрипт использует ключ `-y` для автоматического подтверждения — убедитесь, что это безопасно для вашей среды

### Расписание рекомендуемых обновлений

- **Production-сервера:** 1-2 раза в месяц после тщательного тестирования
- **Staging-сервера:** 1-2 раза в неделю
- **Development-окружения:** еженедельно или по мере необходимости
- **Критические системы:** Только после тщательного тестирования и в плановые окна обслуживания

## apt-mirror-check.sh

Подробная документация: **[APT_MIRROR_CHECK.md](APT_MIRROR_CHECK.md)**

### Описание

`apt-mirror-check.sh` — это интерактивный скрипт для управления APT репозиториями. Скрипт позволяет проанализировать текущие источники обновлений, заменить их на зеркала Yandex для ускорения работы или настроить собственные зеркала вручную.

### Краткий обзор возможностей

- Определение дистрибутива и кодового имени
- Анализ текущих репозиториев из `/etc/apt/sources.list` и `/etc/apt/sources.list.d/`
- Быстрая замена репозиториев на зеркала Yandex
- Ручная настройка пользовательских зеркал
- Автоматическое создание резервных копий перед изменениями
- Восстановление из резервной копии
- Обновление списков пакетов для проверки работоспособности

### Быстрый старт

1. Скачайте скрипт и сделайте его исполняемым:
```bash
chmod +x apt-mirror-check.sh
```

2. Запустите с правами root:
```bash
sudo ./apt-mirror-check.sh
```

3. Выберите действие из интерактивного меню:
   - Заменить на зеркала Yandex
   - Настроить свои зеркала
   - Обновить списки пакетов
   - Восстановить из резервной копии

### Поддерживаемые дистрибутивы

Скрипт автоматически определяет и настраивает зеркала для:
- **Debian** (и производные) — `http://mirror.yandex.ru/debian/`
- **Ubuntu** (и производные) — `http://mirror.yandex.ru/ubuntu/`
- **Linux Mint** — `http://mirror.yandex.ru/linuxmint-packages/`
- **Pop!_OS**
- **elementary OS**
